# 公众号写作发布 Skill（通用社区版）

> ⚠️ **首次使用必读**：
> - 如果你想**让我帮你写文章**：直接说"帮我写一篇XXX"，我全程帮你搞定，最后导出 Markdown
> - 如果你**已经有写好的 .md 文件**：运行 `python wechat_publish.py`，选 B 导出到 WeMark

---

## 快速开始

**你想让我帮你写文章？** 直接说：
```
"帮我写一篇关于XX的公众号文章，品牌是XX，产品是XX"
```
→ 我帮你写 → 写完导出 Markdown → 你复制到 WeMark 发布

---

**你有写好的 .md 文件？** 运行：
```powershell
# 1. 修改 config.yaml 中的 article.file 为你的文章路径
# 2. 运行脚本
& python wechat_publish.py
# 3. 选 B 导出 Markdown → 复制到 WeMark 发布
```

---

**你想一键自动发布到草稿箱？** 运行：
```powershell
# 1. 首次配置（回答4个问题）
& python wechat_publish.py --setup

# 2. 运行脚本
& python wechat_publish.py
# 3. 选 A 自动发布 → 登录草稿箱发布
```

---

## 必填配置项（config.yaml，仅4项）

> 其余全部有默认值，不填也能跑。

| 配置项 | 说明 | 获取方式 |
|--------|------|----------|
| `brand.name` | 品牌名称 | 直接填 |
| `brand.product` | 产品名称 | 直接填 |
| `wechat.appid` | 微信公众号AppID | 公众号后台 → 设置与开发 → 基本配置 |
| `wechat.secret` | 微信公众号AppSecret | 同上（需管理员验证） |

## 工作模式

### 入口一：让我帮你写文章（最推荐）
```
用户说"帮我写一篇XX文章，品牌XX，产品XX"
  → 确认选题和场景
  → 写文章
  → 执行6项合规自检
  → 规划配图方案
  → 整合成Markdown
  → 选 A → 一键发布到草稿箱
  → 选 B → 导出Markdown，让用户手动复制到WeMark
```

### 入口二：用户已有 .md 文件
```
用户已有写好的 .md 文件
  → 修改 article.file 为文件路径
  → 运行 python wechat_publish.py
  → 选 B → 导出Markdown
  → 用户复制到WeMark，手动替换图片
```

### 入口三：一键自动发布（需先配置API）
```
用户运行 python wechat_publish.py --setup 配置API（首次）
  → 修改 article.file 为文件路径
  → 运行 python wechat_publish.py
  → 选 A → 自动发布到草稿箱
```
