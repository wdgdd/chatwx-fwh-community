# 公众号写作发布 Skill（通用社区版）

> ⚠️ **首次使用必读**：
> - 如果你想**让我帮你写文章**：直接说"帮我写一篇XXX"，我全程帮你搞定
> - 如果你**已经有写好的 .md 文件**：运行 `python wechat_publish.py`，选 B 导出 Markdown，复制到任意编辑器发布

---

## 快速开始

**你想让我帮你写文章？** 直接说：
```
"帮我写一篇关于XX的公众号文章，品牌是XX，产品是XX"
```
→ 我帮你写 → 写完导出 Markdown → 你复制到 Markdown 编辑器发布

推荐编辑器：https://md.doocs.org/

---

**你有写好的 .md 文件？** 运行：
```powershell
# 1. 修改 config.yaml 中的 article.file 为你的文章路径
# 2. 运行脚本
& python wechat_publish.py
# 3. 选 B 导出 Markdown → 复制到编辑器发布
```

---

**你想一键自动发布到草稿箱？** 运行：
```powershell
# 1. 首次配置（回答4个问题）
& python wechat_publish.py --setup

# 2. 运行脚本
& python wechat_publish.py
# 3. 选 A 自动发布 → 登录草稿箱发布
```

## 第一步：写文章

### 写作要求
1. **开头**：用真实故事/案例开场，第一人称或第三人称均可，制造代入感和共鸣
2. **结构**：分3-4个层次，不要平铺直叙
3. **产品融入**：作为"解决方案"出现，不硬推，让产品成为解决问题的选项
4. **数据**：用真实数据说话（标准号、营养成分、价格），让读者自己判断
5. **结尾**：有行动指引或情感收尾
6. **文风**：口语化但有质感，有画面感、有细节，不假大空、不自卖自夸、不模板化

---

## 第二步：合规自检（写完必须执行）

> 详见 `REFERENCES/workflow.md` 中的完整自检清单。核心6项如下：

| # | 检查项 | 规则 | 红灯示例 |
|---|--------|------|---------|
| 1 | **数据来源** | 所有具体数字必须有出处或标注"据记载" | "XX年全国第一"无出处 |
| 2 | **措辞合规** | 避免恐吓性用词、绝对化表述 | "致癌物"、"一定会得病" |
| 3 | **不贬低同行** | 对比用客观参数，不用情绪化贬低 | "XX都有毒"、"其他品牌都是垃圾" |
| 4 | **品牌声明真实** | 认证、0添加等需和资质文件一致 | 夸大或虚构认证 |
| 5 | **对比公平** | 竞品对比用可查参数，不带主观判断 | 没有依据的"普通品约XXmg" |
| 6 | **内容禁区** | 不触碰你定义的品类禁区 | — |

## 第三步：配图方案

### 图片规范
- 封面图：900×383px（2.35:1比例）
- 文内图：1080×720px（3:2比例）最佳
- 文件大小：≤2MB（建议500KB-1.5MB）
- 分辨率：≥1920px宽

### 配图数量与位置
每篇文章 3-5 张图：

| 位置 | 类型 | 说明 |
|------|------|------|
| 封面 | 氛围/产品 | 公众号展示用，2.35:1 |
| 开头 | 场景图 | 配合故事开场 |
| 中间（1-2张） | 产品/知识 | 产品图、认证标志、数据表 |
| 结尾 | 氛围图 | 收尾呼应主题 |

### 图片来源
1. **现有素材库**（优先使用，零风险）
2. **免费图库**：Unsplash / Pixabay / Pexels（英文关键词效果更好）

### 配图裁剪指引
| 用途 | 推荐比例 |
|------|---------|
| 封面图 | **2.35:1**（900×383px） |
| 文内场景图 | **3:2**（1080×720px最佳） |
| 产品特写 | 1:1 方图或 4:3 |

## 第五步：发布

### 方式 A：一键发草稿箱（需配置 API）
```
python wechat_publish.py
# 选 A → 自动发布到草稿箱
```

### 方式 B：导出 Markdown（通用）
不需要 API：
```
python wechat_publish.py
# 选 B → 导出 Markdown
```
然后：
1. 打开任意 Markdown 编辑器（如 https://md.doocs.org/）
2. 粘贴 Markdown 内容
3. 按图片序号替换图片
4. 复制全文 → 公众号后台粘贴发布

### 发布失败兜底
**IP白名单报错（errcode 40164）**：
- 用 **Markdown 编辑器** 手动发布
- 或登录公众号后台 → 基本配置 → IP白名单 → 添加你的 IP

**图片上传失败**：
- 检查图片大小是否≤2MB，超过则压缩
- 确认图片格式为 jpg/png/gif

## 排版主题系统

支持6套主题模板，在 `config.yaml` 中切换：

| 主题名 | 名称 | 推荐文章类型 |
|--------|------|-------------|
| `simple` | 简约干净主题 | 知识科普、数据对比 |
| `center` | 居中排版主题 | 情感故事、短篇金句 |
| `decoration` | 装饰线条主题 | 品牌推广、礼盒介绍 |
| `prominent` | 大标题主题 | 深度观点、行业洞察 |
| `warm` | 温暖橙红主题 | 节日节点、下午茶时光 |
| `green` | 绿色健康主题 | 健康生活、日常场景 |

## 工作模式

### 入口一：让我帮你写文章（最推荐）
```
用户说"帮我写一篇XX文章，品牌XX，产品XX"
  → 确认选题和场景
  → 写文章
  → 执行6项合规自检
  → 规划配图方案
  → 整合成Markdown
  → 选 A → 一键发布到草稿箱
  → 选 B → 导出Markdown，让用户复制到编辑器发布
```

### 入口二：用户已有 .md 文件
```
用户已有写好的 .md 文件
  → 修改 article.file 为文件路径
  → 运行 python wechat_publish.py
  → 选 B → 导出Markdown
  → 用户复制到编辑器（如 https://md.doocs.org/）发布
```

### 入口三：一键自动发布（需先配置API）
```
python wechat_publish.py --setup 配置API（首次）
  → 运行 python wechat_publish.py
  → 选 A → 自动发布到草稿箱
  → 发布失败 → 提示IP白名单/手动编辑器方案
```
