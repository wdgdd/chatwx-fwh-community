# 公众号写作发布 Skill（通用社区版）

> ⚠️ **首次使用必读**：
> - 如果你想**让我帮你写文章**：直接说"帮我写一篇XXX"，我全程帮你搞定
> - 如果你**已经有写好的 .md 文件**：运行 `python wechat_publish.py`，选 B 导出 Markdown，复制到任意编辑器发布

---

## 快速开始

**你想让我帮你写文章？** 直接说：
```
"帮我写一篇关于XX的公众号文章，品牌是XX，产品是XX"
```
→ 我帮你写 → 写完导出 Markdown → 你复制到 Markdown 编辑器发布

推荐编辑器：https://md.doocs.org/

---

**你有写好的 .md 文件？** 运行：
```powershell
# 1. 修改 config.yaml 中的 article.file 为你的文章路径
# 2. 运行脚本
& python wechat_publish.py
# 3. 选 B 导出 Markdown → 复制到编辑器发布
```

---

**你想一键自动发布到草稿箱？** 运行：
```powershell
# 1. 首次配置（回答4个问题）
& python wechat_publish.py --setup

# 2. 运行脚本
& python wechat_publish.py
# 3. 选 A 自动发布 → 登录草稿箱发布
```

## 第一步：写文章

### 写作要求
1. **开头**：用真实故事/案例开场，第一人称或第三人称均可，制造代入感和共鸣
2. **结构**：分3-4个层次，不要平铺直叙
3. **产品融入**：作为"解决方案"出现，不硬推，让产品成为解决问题的选项
4. **数据**：用真实数据说话（标准号、营养成分、价格），让读者自己判断
5. **结尾**：有行动指引或情感收尾
6. **文风**：口语化但有质感，有画面感、有细节，不假大空、不自卖自夸、不模板化

### 参考已发布文章风格
> 以下为示例，替换为你的实际文章作为标杆。

## 第三步：配图方案

### 图片规范
- 封面图：900×383px（2.35:1比例）
- 文内图：1080×720px（3:2比例）最佳
- 文件大小：≤2MB（建议500KB-1.5MB）
- 分辨率：≥1920px宽

### 配图数量与位置
每篇文章 3-5 张图：

| 位置 | 类型 | 说明 |
:|------|------|------|
:| 封面 | 氛围/产品 | 公众号展示用，2.35:1 |
:| 开头 | 场景图 | 配合故事开场 |
:| 中间（1-2张） | 产品/知识 | 产品图、认证标志、数据表 |
:| 结尾 | 氛围图 | 收尾呼应主题 |

### 图片来源
1. **现有素材库**（优先使用，零风险）
   ```
   {{article_archive_path}}/assets/          # 你的图片素材目录
   ├── cover-example.jpg                     # 封面图示例
   ├── scene-example.jpg                     # 场景图示例
   ├── product-example.png                   # 产品图示例
   └── footer-brand.gif                      # 品牌结尾图（必用）
   ```

2. **免费图库**（场景氛围图，英文搜索效果更好）
   | 网站 | 网址 | 说明 |
   |------|------|------|
   | **Unsplash** | https://unsplash.com | 推荐直接用，无需API Key |
   | Pixabay | https://pixabay.com/zh/ | 需注册获取API Key，免费每天500张 |
   | Freepik | https://www.freepik.com/ | 注意部分需标来源 |
   | Pexels | https://www.pexels.com/zh-cn/ | 中文界面 |

### 搜索关键词模板

| 场景 | 中文搜索意图 | 推荐英文关键词 |
:|------|------------|--------------|
:| 封面/氛围 | 品牌调性图 | `coffee lifestyle` / `gift box` |
:| 办公/日常 | 白领/职场场景 | `office coffee break` / `work from home` |
:| 送礼 | 节日送礼氛围 | `gift wrapping` / `chinese new year gift` |
:| 产品 | 产品展示 | 直接用本地素材（产品实拍图） |
:| 认证 | 品质认证 | 直接用本地素材（认证标志图） |

> 请根据你的产品品类制定自己的关键词表。英文关键词在 Unsplash/Pixabay 搜索效果更好。

### 下载与压缩
- **优先用 Unsplash 直链**（无需API Key）：
  在 https://unsplash.com/s/photos/关键词 找图，复制照片URL中的图片ID，拼接到 `https://images.unsplash.com/photo-{ID}?w=1920&q=80`
- **Pixabay API 降级**（需API Key）：`https://pixabay.com/api/?key={key}&q={关键词}&image_type=photo`
- 超过2MB用 https://tinypng.com 压缩
- 保存到 `{{article_archive_path}}/assets/第X篇配图/`

### 配图裁剪指引
| 用途 | 推荐比例 | 工具建议 |
:|------|---------|--------|
:| 封面图 | **2.35:1**（900×383px） | WPS图片裁剪/截图工具/PS |
:| 文内场景图 | **3:2**（1080×720px最佳） | 同上 |
:| 产品特写 | 1:1 方图或 4:3 | 同上 |

> 裁剪时优先保留画面主体，避免重要内容被切掉。文内图建议加边框或留白。

## 第四步：整合交付

### 输出文件
一个完整的Markdown文件（纯正文 + 图片插入标注），结构如下：

```markdown
# {{临时标题（发布时再改）}}

[📎插入图片：封面图文件名.jpg]

正文第一段（故事/场景开场）...

[📎插入图片：场景图文件名.jpg]

正文（知识/分析部分）...

[📎插入图片：产品图/认证标志.png]

正文（产品融入/品质说明）...

[📎插入图片：氛围收尾图文件名.jpg]

结尾（情感收尾）

点这里了解详情 → {{你的小程序/商城链接}}

---

{{你的品牌名}} · {{你的产品名}}

{{你的品牌承诺，例如：XX标准起草单位 | 物理冷榨 | 0添加}}
```

### 文末规则
- **不单独列商品/价格**：文末不上商品推荐列表，用情感收尾 + 小程序链接引导
- **商城链接**：从 `config.yaml` 读取 `brand.miniprogram_link`
- **结尾统一追加**：脚本会自动在文末追加品牌结尾图（`brand.footer_image`），不需要在Markdown里手写

### 文件命名
`{{article_archive_path}}/XX-标题.md`

## 第五步：脚本发布到草稿箱

### 发布失败兜底
脚本发布失败有两种常见情况：

**情况1：IP白名单报错（errcode 40164）**
- 原因：当前IP不在公众号后台API白名单里
- 解决：用 **Markdown 编辑器** 手动发布：
  1. 复制 Markdown 文件内容到任意编辑器（如 https://md.doocs.org/）
  2. 配图从 `{{article_archive_path}}/assets/第X篇配图/` 目录选取
  3. 改标题后发布

**情况2：图片上传失败**
- 检查图片大小是否≤2MB，超过则压缩
- 确认图片格式为 jpg/png/gif
- 可尝试手动在公众号后台先上传图片

> **一键加IP白名单（永久解决）**：登录微信公众平台 → 设置与开发 → 基本配置 → IP白名单 → 添加当前IP。查IP：访问 https://myip.ipip.net

### 用户操作
发布到草稿箱后，用户只需：
1. 登录公众号后台 → 草稿箱
2. 打开最新草稿
3. 修改标题（序号 → 正式标题）
4. 确认排版和图片
5. 发布

## 排版主题系统

支持6套主题模板，在 `config.yaml` 中切换：

| 主题名 | 名称 | 推荐文章类型 | 说明 |
:|--------|------|-------------|------|
:| `simple` | 简约干净主题 | 知识科普、数据对比、干货 | 默认主题，排版清爽 |
:| `center` | 居中排版主题 | 情感故事、短篇金句 | 段落居中对齐 |
:| `decoration` | 装饰线条主题 | 品牌推广、礼盒介绍 | 标题带装饰性线条 |
:| `prominent` | 大标题主题 | 深度观点、行业洞察 | 标题加大加粗 |
:| `warm` | 温暖橙红主题 | 节日节点、下午茶时光 | 暖色调，适合情感内容 |
:| `green` | 绿色健康主题 | 健康生活/日常场景 | 暖色调，适合烟火气/产品类 |

> 选主题原则：情感驱动 → `warm`；数据/知识型 → `simple`；日常默认 → `simple`（已设为默认）

**主题配置方式**（在 `config.yaml` 中）：
```yaml
theme:
  name: "green"  # 主题名
  # 或直接指定颜色（自定义主题）
  # primary_color: "#FF5733"
```

## 文章追踪系统

### 采集节奏
每篇文章发布后记录链接 + 发布时间；每周检查一次数据（阅读量/点赞/评论），更新到追踪表。

### 追踪表字段
| 文章标题 | 链接 | 发布时间 | 阅读量 | 点赞 | 评论摘要 |
:|---------|------|---------|-------|-----|--------|

### 数据分析
用户说"看公众号数据"或"分析文章表现"时：
- 调用 `get_article_stats(publisher, days=7)` 获取最近7天数据
- 调用 `analyze_article_stats(stats)` 分析并给出建议
- 找出表现最好的文章，建议复制其场景框架
